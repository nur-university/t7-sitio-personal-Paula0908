Proyecto 1: Salon de Baile

En este proyecto, mi grupo y yo diseñamos un sistema para representar cómo los procesos ingresan a la zona crítica de una computadora. Para hacerlo más comprensible, utilizamos la analogía de un salón de baile.

En esta representación, la pista de baile simboliza la zona crítica, y las parejas de baile representan la interacción entre procesos y recursos. Cada pareja está compuesta por un miembro de cada grupo: uno representa un proceso y el otro un recurso. Esto refleja cómo un proceso necesita de un recurso específico para ejecutar su tarea dentro de la zona crítica. No pueden formarse parejas con dos procesos o dos recursos, ya que ambos roles son necesarios para una ejecución válida.

Las parejas pueden organizarse al inicio de cada ronda de forma ordenada (por ejemplo, h1 con m1, h2 con m2...) o mezclada, según se desee. El acceso a la pista (zona crítica) se gestiona en turnos: las parejas pueden ingresar por orden de llegada o por prioridad.

La prioridad puede ser modificada manualmente por el usuario durante la simulación: basta con hacer clic sobre alguno de los elementos (por ejemplo, h4), y la pareja correspondiente pasará a ser la siguiente en ingresar a la pista una vez que la pareja actual termine su turno. Esta acción no interrumpe el baile actual, sino que define qué pareja será la siguiente en acceder al recurso.

Una vez que un proceso finaliza su uso del recurso, este último puede ser reutilizado por otro proceso sin importar si ya fue utilizado previamente, representando la reutilización de recursos compartidos en sistemas reales.

El objetivo principal de este proyecto fue representar de manera visual, dinámica y sencilla el funcionamiento de la gestión de procesos y recursos en la zona crítica de una computadora, facilitando así la comprensión de conceptos clave como la concurrencia, la sincronización y la gestión de prioridades.

Proyecto 2: Sistema de Registro para Universidad

En este proyecto, mi grupo y yo desarrollamos un sistema de registro académico para una universidad, utilizando Java para la interfaz y lógica del programa, y MySQL como motor de base de datos.

El sistema permite realizar diversas acciones clave como:

  • Registrar estudiantes y docentes.
  • Asignar materias a los estudiantes.
  • Designar docentes responsables de materias específicas.
  • Consultar y visualizar gráficamente los datos almacenados en la base de datos.

Además, implementamos una funcionalidad que muestra todas las tablas normalizadas de la base de datos, con sus respectivos atributos y relaciones, representadas de forma clara mediante consultas SQL que se visualizan en una JTable.

También desarrollamos un módulo de gráficas dinámicas que representa tanto porcentajes como cantidades, en función de la información recuperada desde la base de datos. Un aspecto importante es que estas gráficas se actualizan en tiempo real: cualquier modificación, eliminación o adición de datos en la base afecta directamente a las visualizaciones, garantizando así la consistencia y actualidad de la información mostrada.

Este proyecto tiene como objetivo principal facilitar la gestión académica y mostrar de manera visual y estructurada cómo interactúan los datos dentro de un sistema educativo, haciendo uso de buenas prácticas tanto en programación como en diseño de bases de datos.

Proyecto 3: Tanquesito

En este proyecto, mi grupo y yo desarrollamos un pequeño vehículo autónomo controlado por Bluetooth utilizando Arduino. El sistema incluía distintos componentes electrónicos y mecánicos que trabajaban de forma conjunta para lograr tanto el movimiento como funciones interactivas y automatizadas.

El vehículo contaba con dos motores conectados a las ruedas principales para desplazarse, y una tercera rueda libre —similar a las de los carritos de supermercado— que brindaba estabilidad sin necesidad de motor.

Incorporamos un módulo Bluetooth que permitía controlar el movimiento del vehículo a través de nuestro teléfono celular, brindando una experiencia de manejo remota e intuitiva. Todo esto se conectaba a través de un protoboard y una placa Arduino, que coordinaba los distintos componentes del sistema.

En la parte frontal del vehículo, instalamos un servo motor que giraba constantemente 180 grados de un lado al otro. Encima del servo colocamos un sensor ultrasónico, permitiendo así un escaneo del entorno en movimiento. Si el sensor detectaba un objeto a una distancia determinada (por ejemplo, 20 cm), se activaba una función especial del vehículo: una torreta lanzadora.

Esta torreta fue construida de forma totalmente creativa, utilizando piezas impresas en 3D, palillos de paleta, ligas elásticas y hojas de papel enrolladas. Funcionaba mediante el un motor (como las ruedas), que permitía lanzar pequeños proyectiles (o dulces, como en nuestras pruebas 😄).

Este proyecto no solo nos permitió aplicar conocimientos de programación y electrónica con Arduino, sino también desarrollar creatividad e ingeniería práctica, integrando sensores, actuadores y control remoto en un sistema funcional e interactivo.
